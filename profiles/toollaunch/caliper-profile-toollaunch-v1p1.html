Abstract

This document extends Caliper 1.1 by defining a Tool Launch profile, allowing users to track learning tool utilization from a central location (e.g. an LMS).

Overview

Design Goals and Rationale

The Caliper Tool Launch Profile is intended to capture the utilization of learning tools from a centralized location, such as an LMS. As such, it is distinct from the Caliper Tool Usage profile as defined in the Caliper 1.1 specification [[!CALIPER-11]], which captures tool usage as reported by the individual tools themselves (i.e. decentralized capture).

The Tool Launch and Tool Usage profile have their own unique set of advantages; the primary advantage of the Tool Launch profile is that it  only requires a single sensor to be deployed (typically in an LMS), whereas the Tool Usage profile requires all tools being measured to implement the Caliper sensor API.

Data captured using the Tool Launch profile can be used to determine which tools were launched within a given timeframe, by whom and in what context. It however can not, as opposed to the Tool Usage profile, provide a measure of whether the user actually interacted with the tool in a manner that the application determines to be its “intended use for learning”.

In certain deployment contexts, using both the Tool Launch and the Tool Usage Profile may be advantageous, as it e.g. allows the tracking of the number of tool launches that do not lead to interactions corresponding to the "intended use for learning".

Use cases

Administrators, instructors and researchers can utilize data collected via the Tool Launch Profile to answer questions such as:

  • Which of the available tools are launched the most or the least?
  • Which courses utilize certain tools the most or the least?
  • Are there demographic variances in the utilization of the available tools?
  • Are there temporal variances in the utilization of the available tools?

Events

Tool Launch Event

The Tool Launch Profile is provisioned with a single ToolLaunchEvent for describing tool launches, typically occuring in the context of an LMS.

The Tool Launch Profile models a Person launching a SoftwareApplication at a given time and in a given context. It also provides the ability to model a "return response", which in the context of LTI [[LTI-13]] typically consists of a redirect URL or a specific response message, as defined in for example LTI Deep Linking [[LTI-DL-20]].

Properties
ToolLaunchEvent inherits all properties defined by its supertype Event. Profile specific property type restrictions are described below:
Property Type Description Disposition
id UUID The emitting application MUST provision the ToolLaunchEvent with a UUID. A version 4 UUID SHOULD be generated. The UUID MUST be expressed as a URN using the form 'urn:uuid:<UUID>' per [[!RFC4122]]. Required
type Term The string value MUST be set to ToolLaunchEvent. Required
actor Person | IRI The Person who initiated the action. The actor value MUST be expressed either as an object or as a string corresponding to the actor's IRI. Required
action Term The action or predicate that binds the actor or subject to the object. The value range MUST be limited to one of the 'Launched' or 'Returned' actions only. Required
object SoftwareApplication | IRI The SoftwareApplication that is the target of the Person's launch activity. The object value MUST be expressed either as an object or as a string corresponding to the SoftwareApplication's IRI. Required
edApp SoftwareApplication | IRI A SoftwareApplication that constitutes part of the tool launch context, in the case of this profile typically an LMS. The edApp value MUST be expressed either as an object or as a string corresponding to the edApp’s IRI. Required
federatedSession LtiSession | IRI The Platform's session, constituting part of the tool launch context. The federatedSession value MUST be expressed either as an object or as a string corresponding to the federatedSession’s IRI. Required when the action is 'Launched', otherwise optional. Workflows that have a specific "return message" component (e.g. LTI Deep Linking response messages) SHOULD provide the federatedSession property and SHOULD populate it's messageParameters field with the message parameters in the response message.

Actions

The Caliper ToolLaunchEvent supports the "Launched" and "Returned" actions.

Property Type Description
Launched http://purl.imsglobal.org/caliper/actions/Launched Get going; give impetus to.
Returned http://purl.imsglobal.org/caliper/actions/Returned Go or come back to place, condition, or activity where one has been before.

Representing Caliper as JSON-LD

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote ToolLaunch Profile JSON-LD context for mapping ToolLaunchEvent terms to IRIs. Caliper message producers MUST reference this context in every Caliper 1.1 ToolLaunchEvent created and serialized for transmission to a target endpoint.

See the Caliper 1.1 specification [[!CALIPER-11]] for detailed requirements regarding of JSON-LD and JSON-LD context handling.
