Abstract

This document extends Caliper 1.1 by defining a Feedback Profile that describes feedback related learning activities.

Overview

Design Goals and Rationale

The Caliper Feedback Profile models a Person providing feedback on a Entity, typically a piece of student work. The profile provides a FeedbackEvent along with 'Commented' and 'Ranked' actions for describing feedback activities. The Feedback entity is used to capture the criteria and content of the feedback.

Events

The Feedback Profile is provisioned with a single a FeedbackEvent for describing feedback related activities.

FeedbackEvent

A Caliper FeedbackEvent models a Person providing feedback on a Entity, typically a Response, DigitalResource, or another Person. A Frame may be referenced using the target property.

Commented

Feedback

Properties
FeedbackEvent inherits all properties defined by its supertype Event. Profile-specific type restrictions are described below:
Property Type Description Disposition
id UUID The emitting application MUST provision the FeedbackEvent with a UUID. A version 4 UUID SHOULD be generated. The UUID MUST be expressed as a URN using the form 'urn:uuid:<UUID>' per [[rfc4122]]. Required
type Term The string value MUST be set to 'FeedbackEvent'. Required
actor Person | IRI The Person who initiated the action. The actor value MUST be expressed either as an object or as a string corresponding to the actor's IRI. Required
action Term The action or predicate that binds the actor or subject to the object. The value range is limited to the 'Commented' and 'Ranked' actions only. Required
object Entity | IRI The Entity that is the target of the feedback. The object value MUST be expressed either as an object or as a string corresponding to the resource's IRI. Required
target Frame | IRI A Frame that represents a particular segment or location within the object. The target value MUST be expressed either as an object or as a string corresponding to the target entity’s IRI. Optional
generated Feedback | IRI The Feedback entity that describes the feedback provided. The object value MUST be expressed either as an object or as a string corresponding to the entity's IRI. Optional

Actions

The Caliper FeedbackEvent supports the 'Commented' and 'Ranked' actions.

Term IRI WordNet® Gloss
Commented http://purl.imsglobal.org/caliper/actions/Commented Make or write a comment on.
Ranked http://purl.imsglobal.org/caliper/actions/Ranked Assign a rank or rating to.

Entities

The FeedbackEvent MUST reference a Feedback object that captures the content of the feedback.

Feedback

A Caliper Feedback models a person's reactions, typically textual and/or based on some criteria, to a Entity.

IRI
https://purl.imsglobal.org/caliper/Feedback
Supertype
Entity
Properties
Feedback inherits all properties defined by its supertype Entity. Feedback is also provisioned with one additional property, reaction. Profile-specific type restrictions are described below:
Property Type Description Disposition
id IRI A valid IRI MUST be specified. The IRI MUST be unique and persistent. The IRI SHOULD also be dereferenceable, i.e., capable of returning a representation of the resource. A URI employing the URN scheme MAY be provided in cases where a Linked Data friendly HTTP URI is either unavailable or inappropriate. Required
type Term The string value MUST be set to the Term 'Feedback'. Required
name string A string value comprising a word or phrase by which the Feedback is known. Optional
description string A string value comprising a brief, written representation of the Feedback. Optional
feedbackTarget Entity | IRI The Entity which received the feedback. The feedbackTarget value MUST be expressed either as an object or as a string corresponding to the feedbackTarget resource’s IRI. Optional
feedbackText string A string value comprising a plain text of the feedback given. Optional
scale Scale | IRI The Scale which was used to measure the feedbackTarget by some criteria. The scale value MUST be expressed either as an object or as a string corresponding to the scale resource’s IRI. Optional
scaleSelection string A string value representing the selected value for the feedback from the scale. Optional
dateCreated DateTime An ISO 8601 date and time value expressed with millisecond precision that describes when the Feedback was created. The value MUST be expressed using the format YYYY-MM-DDTHH:mm:ss.SSSZ set to UTC with no offset specified. Optional
dateModified DateTime An ISO 8601 date and time value expressed with millisecond precision that describes when the Feedback was last changed or modified. The value MUST be expressed using the format YYYY-MM-DDTHH:mm:ss.SSSZ set to UTC with no offset specified. Optional
extensions Object A map of additional attributes not defined by the model MAY be specified for a more concise representation of the Feedback. Optional

JSON-LD

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Feedback Profile JSON-LD context for mapping FeedbackEvent terms to IRIs. Caliper message producers MUST reference this context in every Caliper 1.1 FeedbackEvent, Feedback, Scale, and/or LikertScale describe created and serialized for transmission to a target endpoint.

See the Caliper 1.1 specification [[!CALIPER-11]] for a discussion of JSON-LD and JSON-LD context handling.
