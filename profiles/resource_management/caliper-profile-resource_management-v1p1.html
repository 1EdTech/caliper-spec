Abstract

This document extends Caliper 1.1 by defining a Resource Management Profile that describes actions related to managing entities.

Overview

Design Goals and Rationale

The Caliper Resource Management Profile models a Person managing an Entity. The profile provides a ResourceManagementEvent with several actions describing how the resource is being managed: Archived, Copied, Created, Deleted, Described, Modified, Published, Restored, Retrieved, and Unpublished.

User Stories

The Resource Management Profile supports the following learner-centric user stories:

As a content developer, I would like to measure the interval between resource availability and learner engagement with learning resources in order to gain insights into both the administrative impact on the learning process and the timing decisions of learners who are expected to ingest the materials.
As an instructor, I would like to measure engagement between different revisions of learning resources over time in order to help determine what content is most effective.

Events

The Resource Management Profile is provisioned with a single a ResourceManagementEvent for describing the management of entities.

ResourceManagementEvent

A Caliper ResourceManagementEvent models a Person managing an Entity.

Term
ResourceManagementEvent
IRI
https://purl.imsglobal.org/caliper/ResourceManagementEvent
Supertype
Event
Properties
The ResourceManagementEvent inherits all properties defined by its supertype Event, of which id, type, actor, action, object, and eventTime are required. Additional profile-specific type and value restrictions are described below:
Property Type Description Disposition
type Term The string value MUST be set to ResourceManagementEvent. Required
actor Person | IRI The Person who initiated the action. The actor value MUST be expressed either as an object or as a string corresponding to the actor's IRI. Required
action Term The action or predicate that binds the actor or subject to the object. The value range is limited to the Archived, Copied, Created, Deleted, Described, Modified, Published, Restored, Retrieved, and Unpublished actions only. Required
object Entity | IRI The Entity that is being managed. The object value MUST be expressed either as an object or as a string corresponding to the resource's IRI. Required
generated Entity | IRI The Entity that was generated by the Copied action. The object value MUST be expressed either as an object or as a string corresponding to the resource's IRI. Required (Copied action only)

Entities

The ResourceManagementEvent does not add any new entities.

Expressing Caliper as JSON-LD

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Resource Management Profile JSON-LD context for mapping ResourceManagementEvent terms to IRIs. Caliper message producers MUST reference this context in every Caliper 1.1 ResourceManagementEvent describe created and serialized for transmission to a target endpoint.

See the Caliper 1.1 specification [[!CALIPER-11]] for a discussion of JSON-LD and JSON-LD context handling.
