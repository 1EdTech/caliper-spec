Abstract

This document extends Caliper 1.1 by defining a Resource Management Profile that describes actions related to managing digital resources.

Overview

Design Goals and Rationale

The Caliper Resource Management Profile models a Person managing a DigitalResource. The profile provides a ResourceManagementEvent with several actions describing how the resource is being managed: Archived, Copied, Created, Deleted, Described, Downloaded, Modified, Printed, Published, Restored, Retrieved, Saved, Uploaded, and Unpublished.

User Stories

The Resource Management Profile supports the following learner-centric user stories:

  • As a content developer, I would like to measure the interval between resource availability and learner engagement with learning resources in order to gain insights into both the administrative impact on the learning process and the timing decisions of learners who are expected to ingest the materials.
  • As an instructor, I would like to measure engagement between different revisions of learning resources over time in order to help determine what content is most effective.

Caliper JSON-LD Examples

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Resource Management Profile JSON-LD context for mapping ResourceManagementEvent terms to IRIs. Caliper message producers MUST reference the ResourceManagement Profile 1.1-extension context in every Event subtype described in this profile. Likewise, Caliper message producers MUST reference the ResourceManagement Profile 1.1-extension context whenever an Entity subtype described in this profile is emitted on its own as a describe.

See the Caliper 1.1 specification [[!CALIPER-11]] for a discussion of JSON-LD and JSON-LD context handling.
