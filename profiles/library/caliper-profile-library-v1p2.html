Abstract

This document extends Caliper 1.2 by defining a Library Profile that describes usage of library facilities and circulation services.

Overview

Design Goals and Rationale

The Caliper Library Profile models a Person, typically a learner, making use of library facilities and circulation services. The profile provides a LibraryUseEvent for recording use of library facilities, a ResourceUseEvent for describing the loaning of library-owned or licensed assets to library patrons, and a ParticipationEvent for tracking attendance at library-hosted meetings and activities.

User Stories

The Library Profile supports the following user stories:

  • As a librarian,
    • As a librarian, I would like to understand the extent to which library spaces are used for course-assigned activities.
    • As a librarian, I would like to control the level of specificity of the identity or interaction the library enters into the learner record store.
  • As a student,
    • I would like to gauge my viewing of course readings and use of library resources in comparison to my peers.
    • I would like to gauge my use of library services in comparison to my peers.
    • I would like to have a record of the library resources I have used.
  • As a researcher, librarian, instructor, or student,
    • I would like to determine if there is a correlation between student viewing of course readings or use of library resources and measures of student success.
    • I would like to determine if there is a correlation between student use of library spaces and measures of student success.
    • I would like to determine if there is a correlation between student engagement with library services and measures of student success.
Additional user stories can found in M. Oakleaf, Library Integration in Institutional Analytics (November, 2018).

Serializing the Model

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Library Profile JSON-LD context for mapping LibraryUseEvent and ResourceUseEvent terms to IRIs. Caliper message producers MUST reference the LibraryUse Profile 1.2-extension context in every Event subtype described in this profile. Likewise, Caliper message producers MUST reference the LibraryUse Profile 1.2-extension context whenever an Entity> subtype described in this profile is emitted on its own as a describe.

See the Caliper 1.2 specification [[!CALIPER-12]] for a discussion of JSON-LD and JSON-LD context handling.
